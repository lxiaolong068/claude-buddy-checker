## 方案 A：Retro CRT Terminal（复古 CRT 显示器）

**设计运动**：Retrofuturism + CRT Aesthetic（复古未来主义 + 阴极射线管美学）

**核心原则**：
1. 一切元素模拟老式 CRT 终端的视觉效果，包括扫描线、磷光余晖和轻微的屏幕弯曲
2. 信息以"逐字打印"的方式呈现，模拟终端输出的节奏感
3. 色彩极度克制，仅使用绿色/琥珀色磷光色作为前景

**色彩哲学**：
- 背景：纯黑 `#0a0a0a` 模拟关闭的 CRT 屏幕
- 前景：磷光绿 `#33ff33` 或琥珀色 `#ffb000`，模拟早期终端
- 高亮：白色 `#ffffff` 仅用于极少数强调
- 情感意图：唤起 80 年代黑客电影的神秘感和技术浪漫

**布局范式**：
- 全屏单列布局，模拟一个终端窗口
- 顶部是固定的"终端标题栏"（带假的关闭/最小化按钮）
- 内容区域是一个"滚动终端"，所有交互都以命令行形式呈现

**签名元素**：
1. CRT 扫描线叠加层（CSS `repeating-linear-gradient`）
2. 文字的磷光余晖效果（`text-shadow` 多层绿色/琥珀色光晕）
3. 屏幕轻微的桶形失真（`border-radius` + 微妙的 `perspective`）

**交互哲学**：
- 用户"输入命令"而非"点击按钮"——输入框模拟 `>_` 提示符
- 结果以逐行打字动画呈现
- 鼠标悬停时元素产生轻微的"信号干扰"闪烁

**动画**：
- 打字机效果：文字逐字符出现，带光标闪烁
- 页面加载时模拟"开机"效果（屏幕从中心一条线扩展为全屏）
- 稀有度揭示时产生"信号干扰"（glitch）效果

**字体系统**：
- 主字体：`JetBrains Mono` 或 `Fira Code`（等宽字体，模拟终端）
- 标题：同上，但加粗 + 更大字号
- 绝不使用衬线体或非等宽字体

## 方案 B：Cyberpunk Data Terminal（赛博朋克数据终端）

**设计运动**：Cyberpunk UI + Sci-Fi HUD（赛博朋克 + 科幻抬头显示器）

**核心原则**：
1. 界面模拟一个高科技数据分析终端，带有多个"面板"和"数据流"
2. 大量使用半透明面板、发光边框和几何装饰线条
3. 信息密度高，但通过层次分明的发光效果引导视线

**色彩哲学**：
- 背景：深蓝黑 `#0d1117` 到 `#010409`
- 主色：电光蓝 `#58a6ff` + 霓虹青 `#00ffcc`
- 警告/稀有：霓虹粉 `#ff6eb4` 用于 Legendary，金色 `#ffd700` 用于 Shiny
- 情感意图：高科技、未来感、数据可视化的专业感

**布局范式**：
- 非对称的多面板布局，模拟科幻电影中的操作台
- 左侧是"输入面板"，右侧是"数据输出面板"
- 底部是"状态栏"，显示系统信息
- 面板之间用发光的分割线连接

**签名元素**：
1. 发光边框（`box-shadow` 多层霓虹光晕 + `border` 半透明色）
2. 背景中缓慢移动的"数据矩阵"粒子效果
3. 六边形或菱形的装饰元素作为视觉锚点

**交互哲学**：
- 面板在鼠标悬停时"激活"（边框发光增强）
- 数据以"扫描"动画呈现（从上到下的光条扫过）
- 按钮有明显的"充能"按压效果

**动画**：
- 面板加载时从边缘"滑入"并伴随发光效果
- 属性雷达图以动画方式逐轴展开
- 稀有度揭示时整个界面短暂变色（如 Legendary 时全屏闪金光）

**字体系统**：
- 标题：`Orbitron` 或 `Rajdhani`（几何感强的无衬线体）
- 正文/数据：`JetBrains Mono`（等宽，数据展示）
- 标签：`Inter` 细体，全大写，字间距加大

## 方案 C：Minimal Hacker Terminal（极简黑客终端）

**设计运动**：Swiss Minimalism + Terminal Aesthetic（瑞士极简主义 + 终端美学）

**核心原则**：
1. 极度克制的视觉语言——只用文字、线条和留白来构建层次
2. 没有装饰性元素，一切服务于信息传达
3. ASCII 艺术本身就是最大的视觉亮点，不需要额外的视觉噪音

**色彩哲学**：
- 背景：近黑 `#111111`
- 前景：柔和灰白 `#e0e0e0`（非纯白，减少刺眼感）
- 强调色：仅一个——亮绿 `#00ff41` 用于交互元素和关键信息
- 稀有度用不同的 ASCII 符号密度而非颜色来区分
- 情感意图：专业、冷静、"真正的开发者会用的工具"

**布局范式**：
- 垂直居中的窄列布局（max-width: 720px），模拟终端的 80 列限制
- 大量留白，内容块之间用虚线 `---` 分隔
- 没有卡片、没有圆角、没有阴影——纯粹的文字流

**签名元素**：
1. 巨大的 ASCII 艺术 Logo 作为页面顶部的视觉锚点
2. 所有数据用等宽字体的"表格"形式展示（类似 `ls -la` 的输出）
3. 光标闪烁的输入提示符 `$ _`

**交互哲学**：
- 交互极简：一个输入框，一个回车键，结果直接在下方"打印"
- 没有模态框、没有弹窗——一切在同一个"终端流"中完成
- 鼠标悬停仅改变文字颜色为绿色，无其他效果

**动画**：
- 唯一的动画是打字机效果和光标闪烁
- 页面切换无过渡，瞬间完成（模拟终端的即时响应）
- 宠物 ASCII 艺术以逐行打印的方式出现

**字体系统**：
- 全站统一使用 `IBM Plex Mono`（IBM 出品的等宽字体，兼具专业感和可读性）
- 字号层次仅通过大小区分（14px / 16px / 24px），不使用粗体
- 行高 1.6，字间距 0.02em，追求极致的可读性

## 最终选择

我选择 **方案 A：Retro CRT Terminal（复古 CRT 显示器）**，并融合方案 C 的部分极简理念。

理由：CRT 终端风格在视觉上最具辨识度和分享性（社交裂变的关键），磷光绿的配色方案在暗色背景下极其醒目，扫描线和打字机效果能创造强烈的沉浸感。同时借鉴方案 C 的窄列布局和信息克制原则，避免视觉过载。
